# daily-summary-skill 测试样例

## 用例 1：正常日报

输入：

```text
完成：修复训练脚本 bug；新增：跑对比实验；进行中：整理实验结果；quick note：论文讨论章节要用实验表 3。
```

期望章节：

- 今日完成：修复训练脚本 bug。
- 今日推进中：整理实验结果。
- 新增任务与想法：跑对比实验；论文讨论章节使用实验表 3。
- relation 变化：实验结果/实验表 3 支撑论文讨论章节。
- 明日建议：优先跑对比实验并整理结果。

## 用例 2：阻塞任务

输入：

```text
任务：训练被数据清洗阻塞，状态 blocked，优先级 high。
```

期望：

- 今日遇到的问题中说明阻塞原因。
- 明日建议包含先处理数据清洗。
- 不把 blocked 任务标为完成。

## 用例 3：主线进展

输入主线：

```json
["项目", "论文", "健康"]
```

输入变更：

```text
项目完成插件构建修复；论文新增讨论章节任务；健康记录睡眠。
```

期望：

- 各主线进展按项目、论文、健康分组。
- 不使用文件中没有的硬编码主线。

## 用例 4：不确定内容

输入：

```text
今天好像应该整理一下学习资料，但没确定具体内容。
```

期望：

- 不生成已完成项。
- 在新增任务与想法或确认事项中标记待确认。
- 明日建议仅作为候选。
