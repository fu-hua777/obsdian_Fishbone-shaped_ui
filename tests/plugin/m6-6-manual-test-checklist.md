# M6.6 快速输入写入验收清单

## 前置条件

- 已安装并启用 Fishbone Planner 插件。
- Vault 中至少存在一条用户主线，例如“学习”或“项目”。
- 打开 Fishbone Planner 鱼骨画布视图。

## 验收步骤

1. 在画布底部快速输入框输入：`明天 学习 阅读 Transformer 第3章 高优先级`。
2. 点击预览，确认候选卡片展示：
   - 日期为明天。
   - 主线为已有“学习”主线。
   - 优先级为 high/高。
   - 状态默认为 inbox。
3. 点击“确认写入”，确认：
   - 新任务 md 被创建到标准 `PlanningSystem/Tasks/` 路径。
   - frontmatter 中 `source_type: quick-input`。
   - `source_excerpt` 保存原始输入。
   - 鱼骨画布、右侧模块和下方工作台刷新后可看到新任务。
4. 再输入：`后天 不存在主线 测试未分配 低优先级`。
5. 点击预览，确认候选不会自动创建新主线，并提示未匹配到现有主线。
6. 点击“编辑后创建”，确认新建任务弹窗会带入候选标题、日期、优先级、状态和原始输入。
7. 在弹窗中手动选择一个已有主线并创建，确认任务写入成功。
8. 输入包含分号或换行的多条内容，确认系统提示 M6.6 暂按单条候选处理。
9. 点击候选预览关闭按钮，确认不会写入任何新任务文件。

## 通过标准

- 快速输入只在用户点击确认或编辑创建后写入真实 Vault。
- 不存在的主线不会被自动创建。
- 创建后的任务仍是标准 planning-task md。
- `npm run typecheck`、`npm run build` 和 `node scripts/validate-m6-6-quick-input.js` 通过。
