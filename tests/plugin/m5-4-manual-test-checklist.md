# M5.4 手动验收清单：短期分支主线

测试 Vault：

```text
E:\主线规划\主线规划
```

## 准备

1. 运行 `node scripts/create-m5-4-self-test-data.js` 写入 M5.4 自测主线、分支主线和任务。
2. 运行 `npm run build`。
3. 运行 `node scripts/install-m3-plugin.js` 安装最新插件产物。
4. 重启 Obsidian 或禁用后重新启用 `Fishbone Planner`。

## 验收点

1. 打开 `Fishbone Planner: 打开鱼骨时间视图`。
2. 画布中应出现 `M5.4项目` 普通主线。
3. `M5.4短期分支` 应从 `M5.4项目` 分叉出来，显示为有限长度的短主线，而不是普通主线泳道。
4. 分支线应只覆盖 `2026-07-01` 到 `2026-07-07` 的范围。
5. 分支标签应显示分支名称、日期范围和任务数量。
6. 带 `branch_mainline_id: m5-4-branch-ui` 的任务应挂载在分支线上，而不是普通主线上。
7. 日期在 `2026-07-12` 的分支任务应显示在分支结束日期附近，hover 信息提示“显示日期”被限制在分支范围内。
8. 点击“适应窗口”或快捷键 `F` 时，视图范围应包含分支起止日期。
9. 右键父主线折叠后，分支主线和分支任务应同步隐藏；展开后恢复。
10. 隐藏父主线后，分支主线不应残留在画布上。
11. 打开“显示关系”时，分支任务与普通任务的 relation 线仍应连接到正确任务节点。
12. 普通主线创建、改名、删除、排序能力不应受分支数据影响。

## 通过标准

- 普通主线与分支主线视觉层级清晰。
- 分支任务优先挂载到分支主线。
- 分支范围参与时间轴计算。
- 默认 sample-vault 仍不包含任何预设主线。
- 构建和 `scripts/validate-m5-4-branch-mainlines.js` 均通过。
