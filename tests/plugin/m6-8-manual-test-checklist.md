# M6.8 时间 / 天气模块验收清单

## 前置条件

- 已启用 Fishbone Planner 插件。
- 鱼骨画布视图右侧工作台可见。

## 验收步骤

1. 默认未启用天气时，打开鱼骨画布视图，确认“时间 / 天气”模块只显示本地时间和日期。
2. 等待到下一分钟，确认模块时间文本会更新，鱼骨画布不应整体闪烁或重绘。
3. 在插件设置中启用天气，并填写：
   - 地点名称：例如“北京”
   - 纬度：`39.9042`
   - 经度：`116.4074`
   - 温度单位：摄氏度或华氏度
4. 回到鱼骨画布视图，确认模块显示“读取天气缓存中”或“今日暂无天气缓存”，此时不应自动请求网络。
5. 点击“刷新天气”，确认：
   - 模块显示温度、天气描述、风速和获取时间。
   - 写入 `PlanningSystem/WeatherCache/YYYY-MM-DD_weather.json`。
6. 断网或填入无效坐标后点击刷新，确认模块显示失败提示，插件不崩溃，已有缓存不被破坏。
7. 关闭“启用天气”，确认刷新按钮不显示，模块只保留时间。

## 通过标准

- 天气请求只由用户点击“刷新天气”触发。
- 未配置天气时不报错、不请求网络。
- 天气缓存写入本地 Vault 文件。
- `npm run typecheck`、`npm run build` 和 `node scripts/validate-m6-8-time-weather.js` 通过。
