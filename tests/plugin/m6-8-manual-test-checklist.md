# M6.8 时间 / 天气模块验收清单

## 前置条件

- 已启用 Fishbone Planner 插件。
- 鱼骨画布视图右侧工作台可见。

## 验收步骤

1. 打开鱼骨画布视图，确认“时间 / 天气”模块常态显示本地时间、日期和天气缓存状态。
2. 等待到下一分钟，确认模块时间文本会更新，鱼骨画布不应整体闪烁或重绘。
3. 在插件设置中选择“天气地区”，确认地点名称、纬度、经度会自动切换。
4. 选择“自定义”或手动修改地点/经纬度，确认设置可以保存。
5. 回到鱼骨画布视图，点击“同步”，确认：
   - 按钮显示“同步中”，请求结束后恢复为“同步”。
   - 模块时间使用联网返回时间更新。
   - 模块显示温度、天气描述、风速和获取时间。
   - 写入 `PlanningSystem/WeatherCache/YYYY-MM-DD_weather.json`。
6. 断网或填入无效坐标后点击“同步”，确认：
   - 最多约 10 秒后显示失败提示。
   - 按钮恢复可点击，不会一直卡在“同步中”。
   - 已有缓存不被破坏。
7. 清空或填入无效经纬度，确认模块仍显示时间和日期，并提示填写天气经纬度。

## 通过标准

- 天气模块不需要启用开关，始终显示。
- 联网同步由用户点击“同步”触发，同时同步时间和天气。
- 未配置有效经纬度时不报错、不请求网络。
- 联网请求超时后 UI 会恢复，不会一直显示同步中。
- 天气缓存写入本地 Vault 文件。
- `npm run typecheck`、`npm run build` 和 `node scripts/validate-m6-8-time-weather.js` 通过。
