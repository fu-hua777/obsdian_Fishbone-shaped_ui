# M5.2 手动验收清单：任务节点交互与关系线

测试 Vault：

```text
E:\主线规划\主线规划
```

## 准备数据

- 至少 2 条用户主线，例如「项目」「学习」。
- 至少 4 个标准规划任务。
- 至少 2 个任务在同一日期同一主线，用来检查节点错位。
- 至少 2 条 relation：
  - 同主线跨日期关系。
  - 跨主线跨日期关系。

## 验收步骤

1. 打开命令 `Fishbone Planner: 打开鱼骨时间视图`。
2. 确认画布上仍然没有系统默认主线；未分配任务显示在「未分配」泳道。
3. 长按任务节点并拖到另一日期，松手后检查任务 md 的 `date` 已更新。
4. 长按任务节点并拖到另一主线，松手后检查任务 md 的 `mainline` 已更新。
5. 长按任务节点并拖到「未分配」泳道，松手后检查任务 md 的 `mainline` 为 `null`。
6. 普通点击任务节点，确认仍然打开对应任务 md。
7. 勾选和取消勾选任务完成框，确认状态切换不需要点击两次。
8. 右键任务节点，确认可打开任务、编辑任务属性、修改状态、修改优先级、移动到未分配、复制 `task_id` 和路径。
9. 在任务属性弹窗里修改标题、日期、主线、状态、优先级，确认只更新当前任务 md 的 frontmatter。
10. hover 任务节点，确认显示标题、日期、主线、状态、优先级、关系数量和摘录。
11. 确认有 relation 的任务之间显示虚线、箭头和标签。
12. hover 任务节点，确认相关 relation 线高亮。
13. hover relation 线，确认来源任务和目标任务高亮。
14. 右键 relation 线，确认菜单显示关系、来源和目标，并可打开来源/目标任务 md。
15. 点击工具栏「隐藏关系 / 显示关系」，确认 relation 层可开关。
16. 平移画布后，确认任务节点、主线和 relation 线仍然对齐。
17. `Ctrl + 滚轮` 缩放画布后，确认任务节点和 relation 线一起缩放。
18. `Alt + 滚轮` 或在日期轴上 `Ctrl + 滚轮` 调整时间轴密度后，确认 relation 线重新对齐。
19. 长按主线标签调整排序，确认 M5.1 主线排序交互不回退。

## 通过标准

- 任务节点可拖动修改日期和主线。
- 任务节点可右键编辑关键字段。
- 任务 hover 可查看详情。
- relation 可显示、可开关、可 hover 高亮、可右键定位两端任务。
- M5.1 的画布平移、缩放、时间密度缩放和主线排序不回退。
