# M5.1 手动验证清单

测试 Vault：

```text
E:\主线规划\主线规划
```

## 前置条件

- `npm run build` 已通过。
- `node scripts/install-m3-plugin.js` 已执行。
- Obsidian 已重载。
- `Fishbone Planner` 插件已启用。

## 验证步骤

1. 打开命令面板。
2. 运行 `Fishbone Planner: 打开鱼骨时间视图`。
3. 确认顶部出现 `上一周`、`下一周`、`今天`、`显示全部`。
4. 确认默认显示当前周 7 天。
5. 确认没有任务的日期也显示。
6. 确认今天日期列高亮。
7. 点击 `上一周` 和 `下一周`，确认日期窗口切换。
8. 点击 `今天`，确认回到当前周。
9. 点击 `显示全部`，确认能看到任务日期范围。
10. 确认日期区域可横向滚动。
11. 确认任务仍挂载到对应主线和日期。
12. 确认同一日期同一主线有多个任务时，任务节点会上下错位。
13. 确认高/中/低优先级有不同视觉。
14. 确认 `done`、`doing`、`blocked` 等状态有不同视觉。
15. 确认 M5 的主线创建、修改、删除、排序仍可用。
16. 确认任务 checkbox 和主线选择仍可用。

## 不应出现

- 不应只显示有任务的日期。
- 不应自动创建默认主线。
- 不应覆盖已有 `PlanningSystem`。
- 不应引入右侧工作台或底部看板。
