# M6.3-M6.4：今日聚焦、状态看板、本周重点与进度概览计划

## 目标

在 M6.1-M6.2 的数据聚合层和右侧模块容器基础上，把辅助面板从只读摘要推进到可执行工作台。M6.3 负责今日聚焦和状态看板的任务状态写回；M6.4 负责本周重点和进度概览的可扫描信息增强。

## M6.3：今日聚焦与状态看板

- 今日聚焦模块显示今天任务，按优先级、状态和标题稳定排序。
- 今日聚焦任务行支持勾选完成；取消勾选恢复为 `todo`。
- 今日聚焦任务行支持快速选择状态：`todo / doing / done / blocked / canceled / inbox`。
- 点击任务主体仍打开对应任务 md。
- 状态看板模块不只显示数量，还显示待办、进行中、阻塞、已完成的代表任务。
- 状态看板任务同样支持快速改状态，写回当前任务 md frontmatter 后刷新鱼骨视图。

验收标准：

- 右侧模块改状态后，任务 md frontmatter 更新，鱼骨画布重新渲染。
- 写入只通过 `TaskRepository.setTaskStatus()`，不批量修改任务文件。
- 控件点击不会误触发打开任务。

## M6.4：本周重点与进度概览

- Dashboard 聚合层输出 `weekFocusTasks`，统一合并本周高优先级、阻塞、进行中和过期未完成任务。
- Dashboard 聚合层输出 `overdueTasks`，用于提示已经早于今日且未完成/未取消的任务。
- 本周重点模块展示任务原因标记：过期、高优先、阻塞、进行中。
- 主线进度模块显示每条主线完成率，同时显示总数、进行中、阻塞、高优先、过期数量。
- 今日/本周进度模块继续显示完成率，但保留进行中、阻塞、待办信息。

验收标准：

- 本周重点不重复显示同一任务。
- 过期任务即使不在本周，也能进入本周重点列表。
- 主线进度不创建默认主线；未分配任务仍使用虚拟分组。

## 不做

- 不做拖拽式 Kanban 看板。
- 不做快速输入候选任务。
- 不做每日总结生成。
- 不修改任务 schema。
- 不自动扫描真实旧笔记。
