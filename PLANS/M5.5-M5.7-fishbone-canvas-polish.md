# M5.5-M5.7：鱼骨主画布收口计划

## 目标

把鱼骨主画布从“功能可用”推进到“布局清晰、拖拽可信、分支语义明确”的状态。本阶段仍只处理鱼骨主画布，不进入右侧工作台、快速输入、每日总结或底部看板。

## M5.5：节点避让与上下轨道布局

- 任务节点按 `主线/分支 + 上下侧 + 轨道` 进行布局，不再只按同一天任务桶简单错位。
- 同一主线相邻日期的任务节点也参与碰撞检测，避免不同日期任务标签互相重叠。
- 每条主线的上侧和下侧都可以拥有多条纵向轨道，轨道之间保持固定最小间距。
- 单任务日期桶按日期和主线稳定交替分配到上侧/下侧，避免所有单任务都挤在主线一侧。
- relation 布局使用任务节点实际锚点重新计算，节点避让后关系线仍对齐。

验收标准：
- 同一主线、相邻日期的任务节点不会明显互相覆盖。
- 同一天多任务仍能在主线上下两侧错开。
- 缩放后节点位置仍随画布一致变化。

## M5.6：任务拖拽边界与写回规则

- 普通任务拖拽到普通主线时，按落点写回 `date` 和 `mainline`。
- 普通任务拖入分支主线时，写回 `date`、父主线 `mainline`、`branch_mainline_id` 和 `branch_mainline`。
- 分支任务在分支内拖拽时，日期被限制在分支起止日期范围内。
- 分支任务拖回普通主线或未分配泳道时，清除 `branch_mainline_id` 和 `branch_mainline`。
- 无效落点不写入文件，任务节点回到原始显示位置。
- 拖拽提示显示实际将写入的日期、目标主线或目标分支。

验收标准：
- 每次合法拖拽后，只更新当前任务 md 的 frontmatter。
- 写回字段包含 `updated`，并等待 metadata cache 更新。
- 非法拖拽不会留下错误的临时节点位置。

## M5.7：分支标签与平滑连接线

- 分支主线拥有独立的视觉语义，不再像普通任务标签。
- 分支标签层级高于任务节点，避免被分支任务遮挡。
- 分支标签根据分支位于父主线上方/下方自动放到分支线外侧。
- 父主线到分支主线的连接线使用平滑 SVG 曲线，不再使用固定方向直线。
- 分支上方/下方移动后，连接线自动重新选择曲线方向和控制点。
- relation 线使用任务实际锚点，并根据拥挤程度增加轻量弯曲偏移。

验收标准：
- 分支拖到父主线上方或下方，连接线都能正确接到分支线。
- 分支标签不会被分支任务覆盖。
- 关系线、分支线、任务节点同时存在时层级清晰。

## 不做

- 不做右侧工作台。
- 不做快速笔记 / Codex 输入。
- 不做每日总结。
- 不做底部状态看板。
- 不改变标准任务 schema。
- 不创建默认主线。
