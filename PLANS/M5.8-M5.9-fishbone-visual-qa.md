# M5.8-M5.9：鱼骨画布视觉 QA 与验收辅助计划

## 目标

在 M5.5-M5.7 的布局、拖拽和分支线收口基础上，补齐可视化诊断和验收辅助，降低后续修改再次引入“重复线、错位线、标签遮挡”的风险。本阶段仍只处理鱼骨主画布，不进入右侧工作台、快速输入、每日总结或底部看板。

## M5.8：分支视觉一致性

- 短期主线的曲线接入和横向分支线必须由同一个 SVG path 渲染。
- DOM 分支主线只作为透明交互命中区，不再绘制第二条横线或接头点。
- 分支线、透明命中区和透明标签共享同一个 `data-branch-mainline-id`。
- 鼠标悬停或拖拽短期主线/标签时，只高亮对应的 SVG path，便于确认当前交互对象。
- 短期主线标签保持透明、简洁，只显示名称；日期范围保留在 hover title。

验收标准：
- 画布上每条短期主线只有一条可见横线。
- 父主线到短期主线的接入曲线与横线连续，不出现上下错位。
- 悬停标签或分支命中区时，高亮的是同一条分支线。

## M5.9：验收辅助与回归防护

- 新增 M5.8-M5.9 验证脚本，检查单一 SVG path、透明标签、共享 active 状态和旧 DOM 横线禁用。
- 新增手动验收清单，覆盖重载插件、刷新画布、测试短期主线、标签简洁性和悬停高亮。
- 更新 `DEVLOG.md` 记录本阶段变更和仍需人工视觉确认的边界。

验收标准：
- `node scripts/validate-m5-8-9-visual-qa.js` 通过。
- 既有 M5.4、M5.5-M5.7、M5 interaction 校验仍通过。
- `npm run build` 通过，并安装到真实测试 vault。
