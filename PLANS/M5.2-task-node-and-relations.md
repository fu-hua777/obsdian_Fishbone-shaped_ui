# M5.2：任务节点交互与关系线合并阶段计划

## 目标

将原计划中的 M5.2「任务节点交互增强」与 M5.3「relation 关系线」合并为一个阶段：在现有鱼骨画布基础上，让任务节点成为可拖动、可编辑、可感知关系的交互对象，并在同一套坐标模型中渲染任务关系线。

合并原因：

- 任务拖动会改变任务的日期、主线和画布坐标。
- relation 关系线依赖任务节点坐标、锚点和可见性。
- 两者如果分阶段独立实现，会重复改 `fishboneCanvasLayout.ts`、`FishboneTimelineView.ts` 和任务写回逻辑。

## 当前基础

已完成：

- 鱼骨视图已改为可平移、可缩放的 canvas stage。
- 主线由用户通过 UI 创建，不创建默认主线。
- 主线名称固定显示在左侧覆盖层，可编辑、删除、排序。
- 任务节点可显示、勾选完成、打开 md、通过 select 切换主线。
- 画布整体缩放、单主线缩放、时间轴密度缩放已经存在。

当前不足：

- 任务节点不能拖动修改日期。
- 任务节点不能拖动切换主线。
- 任务节点不能右键编辑标题、日期、状态、优先级等字段。
- relation 只存在于任务 frontmatter 中，画布上还没有关系线。
- 布局层没有输出任务节点尺寸、锚点、关系线坐标、命中区域。

## 范围

### 1. 任务节点坐标与命中模型

扩展 `fishboneCanvasLayout.ts`：

- 给 `FishboneCanvasTaskNode` 增加稳定几何信息：
  - `width`
  - `height`
  - `anchorTop`
  - `anchorBottom`
  - `anchorLeft`
  - `anchorRight`
  - `spineAnchor`
- 增加任务查询映射：
  - `taskId -> node`
  - `path -> node`
  - `title -> node`
- 增加画布命中辅助：
  - 根据 client 坐标反推 canvas 坐标。
  - 根据 canvas y 判断目标主线。
  - 根据 canvas x 反推目标日期。
- 保持未分配任务仍能显示在未分配泳道。

### 2. 任务拖动修改日期和主线

在 `FishboneTimelineView.ts` 中新增任务节点 pointer 拖拽：

- 长按任务节点后进入拖动，不影响普通点击打开任务。
- 拖动时显示节点浮动状态。
- 拖动过程中显示目标日期和目标主线的落点提示。
- 松手后：
  - 根据 x 位置更新 `date`。
  - 根据 y 位置更新 `mainline`。
  - 如果落到未分配泳道，则写入 `mainline: null`。
- 拖动取消时回到原位置。
- 拖动后抑制一次 click，避免误打开任务 md。

`TaskRepository` 需要新增：

- `setTaskDate(task, date)`
- `setTaskPriority(task, priority)`
- `setTaskStatus(task, status)`
- `updateTaskFields(task, patch)`

写入规则：

- 只修改当前任务 md 的 frontmatter。
- 每次写入更新 `updated`。
- 不批量改其他任务。
- 不在本阶段维护 `task-index`。

### 3. 任务右键编辑菜单

在任务节点上支持右键菜单：

- 打开任务 md
- 编辑任务属性
- 标记为 todo / doing / done / blocked / canceled
- 修改优先级 high / medium / low
- 移动到未分配
- 复制任务路径或 task_id

任务属性编辑弹窗包含：

- 标题
- 日期
- 主线
- 状态
- 优先级

标题编辑策略：

- 本阶段先更新 frontmatter `title`。
- 可选同步 markdown 一级标题，但必须谨慎处理；如果没有稳定规则，先不改正文标题。

### 4. 任务悬浮详情

任务节点 hover 时显示轻量详情：

- 标题完整文本
- 日期
- 主线
- 状态
- 优先级
- relation 数量
- source excerpt 简短内容

要求：

- 不遮挡正在拖动的任务。
- 不触发画布平移。
- 不影响 checkbox/select 操作。

### 5. relation 解析与关系线布局

新增 relation 解析逻辑：

- 从每个任务的 `relations` 字段读取关系。
- 支持 target 匹配：
  - `task_id`
  - wiki link：`[[标题]]`
  - 文件 path
  - 任务 title
- 找不到目标任务时跳过渲染，并可在开发日志或调试信息中记录。

扩展 `fishboneCanvasLayout.ts` 输出：

- `relationLines: FishboneCanvasRelationLine[]`

每条 relation line 包含：

- source task node
- target task node
- relation type
- direction
- label
- color
- dashed style
- arrow direction
- path points 或 bezier 控制点

### 6. relation 视觉渲染

在 canvas stage 中新增 relation layer：

- layer 顺序：
  - date axis
  - relation layer
  - mainline spine layer
  - task layer
  - label overlay layer
- relation 使用 SVG 或 HTML canvas 均可，但建议先用 SVG：
  - 更容易画虚线、箭头、标签。
  - 更容易随 stage transform 一起缩放。

关系类型样式：

- 普通关联：灰色细虚线
- 前置 / 依赖：带箭头虚线
- 阻塞：红色或警示色虚线
- 参考：弱化虚线
- 支撑 / 影响：带标签虚线

交互：

- hover 关系线时高亮源任务和目标任务。
- hover 任务节点时高亮与它相关的关系线。
- 提供关系线显示开关。

### 7. relation 右键与最小编辑

本阶段优先做显示，不优先做复杂关系编辑。

最小可做：

- 右键关系线：显示来源任务、目标任务、关系类型、label。
- 打开来源任务 md。
- 打开目标任务 md。

暂不做：

- 拖拽创建 relation。
- 关系编辑器。
- 自动推断 relation。

### 8. 任务聚集与避让的第一版

当前任务节点同日同主线只做上下错位。

本阶段增强：

- 同一日期同一主线任务过多时，自动增加纵向偏移。
- 节点之间保持最小间距。
- 超出主线高度时给出紧凑堆叠显示。
- 后续再做聚合展开，不在本阶段做复杂聚合。

## 不做项

- 不做右侧工作台。
- 不做底部看板。
- 不做每日总结模块。
- 不做 task-index 同步。
- 不扫描真实 Vault 旧笔记。
- 不做自动 relation 推断。
- 不做拖拽创建 relation。
- 不做完整任务正文编辑器。
- 不改变标准 task schema。
- 不创建默认主线。

## Obsidian 手动验收

测试 Vault：

```text
E:\主线规划\主线规划
```

需要准备：

- 至少 2 条主线。
- 至少 4 个任务。
- 至少 2 个任务在同一日期同一主线。
- 至少 2 条 relation：
  - 同主线跨日期。
  - 跨主线跨日期。

验收步骤：

1. 打开 `Fishbone Planner: 打开鱼骨时间视图`。
2. 拖动任务到另一日期，确认任务 md 的 `date` 更新。
3. 拖动任务到另一主线，确认任务 md 的 `mainline` 更新。
4. 拖动任务到未分配泳道，确认 `mainline: null`。
5. 右键任务节点修改状态和优先级，确认 frontmatter 更新。
6. hover 任务节点，确认详情浮层显示。
7. 查看 relation 线，确认箭头、虚线、标签显示。
8. 缩放画布，确认任务和 relation 线一起缩放。
9. 调整时间轴密度，确认 relation 线重新对齐。
10. 勾选任务完成，确认 M5 checkbox 行为不回退。
11. 长按拖动主线排序，确认 M5.1 主线交互不回退。

## 风险点

- 任务拖拽与点击打开任务容易冲突，需要 click 抑制。
- 任务拖拽与 checkbox/select 不能互相干扰。
- relation target 匹配可能不稳定，需要明确匹配优先级。
- SVG relation layer 的 pointer event 需要谨慎设置，避免挡住任务节点。
- 任务标题修改是否同步正文标题需要单独决策，本阶段默认只更新 frontmatter。
- 关系线多时可能影响性能，第一版应限制复杂动画。

## 完成标准

- 任务节点可拖动修改日期和主线。
- 任务节点可右键编辑关键字段。
- 任务 hover 可查看详情。
- relations 可在画布上以虚线、箭头、标签形式显示。
- 任务拖动、时间轴缩放、画布缩放后 relation 线能重新对齐。
- 不破坏 M5.1 的画布、主线、缩放、排序交互。
- 构建和 M5.2 验证脚本通过。
