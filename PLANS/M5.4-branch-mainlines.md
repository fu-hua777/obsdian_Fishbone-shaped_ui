# M5.4 分支主线拆分计划

## 背景

长时间任务不再作为一个超宽任务卡片处理，而是允许被表达为“短期分支主线”：它从某条用户主线分叉出来，拥有有限日期范围，并可继续挂载多个小任务。这样保留鱼骨画布的主线结构，也避免长任务挤压普通日期任务。

## M5.4.1 数据模型与兼容读取

- 扩展 `Mainline`：新增 `type`、`parentMainlineId`、`startDate`、`endDate`。
- 兼容旧 `mainlines.json`：没有 `type` 的旧主线一律视为普通主线。
- 支持分支主线 JSON 字段：`type: "branch"`、`parent_mainline_id`、`start_date`、`end_date`。
- 扩展 `PlanningTask`：支持 `branch_mainline` 与 `branch_mainline_id`，用于把小任务挂载到分支主线。
- 写回 `mainlines.json` 时保持普通主线简洁，只给分支主线写入分支字段。

## M5.4.2 分支主线静态渲染

- 普通主线仍由用户创建，分支主线不进入普通主线泳道列表。
- 分支主线按父主线、起止日期和排序渲染为短线段。
- 分支线从父主线脊骨分叉，显示分支名称、日期范围和任务数量。
- 父主线折叠或隐藏时，对应分支同步隐藏。
- 分支起止日期纳入“适应窗口”和时间轴范围计算。

## M5.4.3 分支子任务挂载

- 带 `branch_mainline_id` 或 `branch_mainline` 的任务优先挂载到对应分支主线。
- 分支任务仍保留原任务 md 作为唯一数据源。
- 分支任务若日期超出分支范围，画布显示位置夹取到分支起止范围内，并在 hover 信息中提示显示日期。
- 同一天同一分支的任务继续使用动态日期槽宽和上下交错布局，避免挤压。
- 分支任务使用轻量区分样式，防止与普通主线任务混淆。

## M5.4.4 分支主线编辑入口

- 在任务右键或编辑弹窗中增加“转换为分支主线”的入口。
- 支持选择父主线、起始日期、结束日期、名称和颜色。
- 转换后原长任务可作为分支说明任务保留，或仅作为分支来源引用。
- 支持修改分支主线名称、日期范围、颜色、父主线。
- 支持删除分支主线，并确认是否解除其子任务挂载。

## M5.4.5 分支交互与画布行为

- 分支主线可被选中、拖动调整垂直偏移，但日期端点仍受起止日期控制。
- 分支上的小任务拖拽只允许在分支日期范围内移动。
- 支持从普通任务拖到分支主线上，写入 `branch_mainline_id`。
- 支持从分支拖回父主线或其他普通主线，清除分支字段。
- 关系线需要正确连接普通任务、分支任务和跨分支任务。

## M5.4.6 验收与打磨

- 分支主线视觉需要接近概念图中的“主线分叉”语义，而不是表格行。
- 密集日期、长范围分支、折叠父主线、隐藏父主线、跨分支 relation 均需手动验收。
- 保持默认没有主线，不写死健康、学习、项目等默认分类。
- 通过构建、M5.4 静态校验、手动验收清单和测试 Vault 自测数据验证。
