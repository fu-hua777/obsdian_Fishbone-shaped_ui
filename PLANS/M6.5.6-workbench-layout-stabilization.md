# M6.5.6 工作台布局稳定化

## 目标

M6.5.6 聚焦已经存在的 L 型工作台，不新增新的业务模块。目标是让右侧模块、下方状态工作台和悬浮快速输入在 Obsidian 内稳定可用，并且重启后保持用户调整过的布局。

## 范围

- 右侧模块排序改为 pointer-based 拖拽，避免 Obsidian WebView 中 HTML5 drag/drop 读取自定义 DataTransfer 不稳定。
- 下方三列排序改为 pointer-based 拖拽，任务跨列状态写回仍保留原生拖放。
- 右侧模块高度、下方工作台高度、右侧模块顺序、下方列顺序继续写入 `dashboardState`。
- 排序过程中显示明确的插入线，而不是仅高亮整块区域。
- 右侧区域与下方区域保持 L 型边界：右侧贯穿上下，下方只在画布下方，不覆盖右侧。
- 快速输入框保持独立，固定在鱼骨画布底部中间，不属于右侧或下方模块容器。

## 不做

- 不修改鱼骨画布主线、任务、关系线和短期主线布局。
- 不实现快速输入写入标准任务 md，该能力保留到后续阶段。
- 不引入每日总结、天气或外部 API。

## 验收

- 拖动右侧模块标题，模块顺序立即调整并保存。
- 拖动下方列标题，三列顺序立即调整并保存。
- 拖动任务条到其他状态列，任务 md 状态正确写回。
- 调整右侧模块高度和下方工作台高度后刷新视图，尺寸保持。
- 重启 Obsidian 后，模块顺序、列顺序和高度仍保持。
- `npm run typecheck`、`npm run build` 和 M6 验证脚本通过。
